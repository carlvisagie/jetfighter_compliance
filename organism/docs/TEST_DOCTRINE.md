# KYC Organism Test Doctrine

The test suite must prove the platform can handle real client chaos.

It must test:
- missing documents
- wrong documents
- duplicate documents
- stale documents
- renamed files
- screenshots instead of PDFs
- partial uploads
- corrupted files
- vague client notes
- no company identity
- no primary contact
- no contract requirement
- no policy evidence
- no vendor evidence
- conflicting versions
- payment without intake
- intake without payment
- webhook failure
- interrupted upload
- orphan artifact
- broken chain of custody
- classification uncertainty
- escalation creation
- recovery after restart
- reconstruction from events
- calm customer language
- precise operator language

The platform must prove it can answer:
- What did the client give us?
- What is missing?
- What is probably wrong?
- What is duplicated?
- What is stale?
- What should happen next?
- What can the organism safely fix?
- What requires human judgment?
- What should the client be asked for?
- What should the operator see?
- What must be preserved for audit?

Every test must support the mission:
Upload whatever you already have.
We will organize it.
We will take it from here.
