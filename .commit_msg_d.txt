Phase 4: restore the organism heartbeat — scheduler back online

The organism was clinically dead in production. Three stacked bugs:

  1. start_worker() in services/engine.py had its scheduler-creation code
     commented out as a "stabilization" measure; the function only logged
     "code disabled" even when KYC_SCHEDULERS_ENABLED=true.

  2. _boot_worker() in server.py NEVER CALLED start_worker(). Even if (1)
     had been fixed, the startup hook would still log "worker: skipped"
     forever without invoking the heart.

  3. sweep_queue() (the canonical V1) globbed (DATA / "jobs") instead of
     JOBS, so test monkeypatching of engine.JOBS silently broke -- which
     is why someone had bolted on the test-safe overrides that got
     removed in 14c7ec8. With JOBS as the canonical source, the original
     organism-integrated _process_one() path works in both tests and
     production.

  4. worker_fix.txt at the repo root was a notepad fragment of what
     start_worker should look like, written months ago and never applied.
     Deleted.

Rewrite philosophy:

  - One scheduler per process, gated by KYC_SAFE_MODE / KYC_SCHEDULERS_ENABLED.
  - PER-ORGAN KILL SWITCHES via KYC_DISABLE_<ORGAN> env vars (queue, sla,
    nightly_exports, weekly_digest, compliance_intel, acquisition,
    alerts). An operator can isolate any sick organ from the Render
    Dashboard without a code change. Default is ON.
  - Every scheduler decision (create, organ-add, organ-skip, organ-fail,
    module-register, module-fail, start, start-fail, full-disable) emits
    BOTH a boot-log entry AND a `system` telemetry event into central
    memory. The awareness layer now sees the real heart rate, not env-
    var fiction.
  - Inline imports of services.memory.telemetry so a missing/booting
    memory subsystem cannot prevent scheduler boot.

DiskPersistenceCheck no-ops silently when no disk_persistence signal is
present in the bundle (legacy tests, partial probes). This restores the
old "empty bundle => GREEN" contract that two organism_state tests
depend on, without weakening the production check (production always
ships the signal because DiskPersistenceCollector runs).

Full test suite: 860 passed, 0 failed (was 857/3 before this commit).
