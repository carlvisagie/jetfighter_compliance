# Founding Beta Organism Validation Mode

## Current goal

Validate the organism with **real paperwork**, not vanity metrics.

We are validating:

- Upload-first onboarding
- Evidence intake and classification
- Burden detection (including quiet operational uncertainty)
- Acquisition intelligence
- Contextual knowledge overlay
- Operational guidance
- Telemetry learning loops

## Success metric (temporary)

**Real paperwork submitted** — not clicks, karma, engagement, or impressions.

## Acquisition tuning

**Good prey:** operational burden, implementation questions, vendor/customer pressure, small-business applicability confusion, real paperwork burden.

**Bad prey:** consultants, AMAs, educators, promoters, generic topic chatter, thought experiments.

**Soft operational burden priority:** increase sensitivity to quiet confusion without reopening generic CMMC discussion floods.

**Queue target:** 3–10 quality prospects per discovery cycle.

## Founding beta messaging

Use collaborative, transparent, low-pressure copy. See `services/founding_beta/messaging.py`.

## Telemetry

Track: `beta_upload_started`, `beta_upload_completed`, `upload_file_types`, `evidence_mapping_confidence`, `operator_review_needed`, `onboarding_confusion`, `overlay_helpfulness`, `acquisition_to_upload_conversion`.

Enable/disable: `KYC_FOUNDING_BETA_MODE=true` (default on).
